# Requirements

> *Parent: [[Context]]*

Extract and organize the qualifications and skills from the job posting. Separating required from preferred helps prioritize what to emphasize in your application.

## Rules

- Extract requirements **from the Description** — don't invent requirements that aren't stated.
- Clearly separate required vs. preferred — this drives the Mapping priority.
- Note any implicit requirements (e.g., the posting mentions tools or methods without listing them as requirements).
- Use consistent language so Mapping can reference these directly.
